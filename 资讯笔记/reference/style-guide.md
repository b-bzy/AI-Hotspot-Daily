# 机器之心风格指南

机器之心是国内头部 AI 科技媒体，核心气质：**有深度、有现场感、不标题党、不口水**。
读者是 AI 从业者、研究员和科技爱好者，能消化技术细节，但不喜欢纯学术腔。

---

## 一、文章类型识别

写作前先判断素材属于哪种类型，不同类型对应不同结构模板。

| 类型 | 典型素材特征 | 对应模板 |
|------|-------------|---------|
| **A · 产品/融资快讯** | 单一产品发布、融资公告、功能更新 | 微观场景开篇 → 新闻锚点 → 技术深度 → 市场落地 |
| **B · 战略/研究综述** | 公司多篇论文、战略布局、赛道进入信号 | 宏观行业转折开篇 → 战略信号 → 多项工作串讲 → 人物/团队收尾 |
| **C · 单篇论文解读** | 单篇 arxiv/研究报告 | 问题痛点开篇 → 方法核心 → 实验数据 → 行业意义 |

大多数稿件属于 A 或 B。C 类通常只在论文本身足够重磅时单独成文。

---

## 二、开篇模式（两种，按文章类型选择）

### 开篇 A：微观场景钩子（适合产品/融资类）

用具体物理场景或操作细节切入，点出技术痛点。好的开篇让读者「哦，这个问题我也感受过」。

> 问题场景（细节丰富）→ 分析根因 → 引出解法方向

**示例**（新智具身）：
> 当机器人执行精细操作时，真正的考验往往始于接触的瞬间：USB 插头看似对准接口，却在插入的瞬间因毫厘之差而被卡住……

### 开篇 B：宏观行业转折（适合战略/综述类）

先点出行业大趋势的拐点，再引出「一个容易被忽视的玩家」或「一个反常识的方向」。

> 行业竞争逻辑正在改变 → 一个低调但有独特优势的玩家出现 → 释放明确信号

**示例**（三星 AI 战略）：
> 过去几年，大模型竞争主要发生在 AI 公司之间。但随着 AI 开始进入真实设备与物理世界，竞争逻辑正在发生变化……最近，机器之心注意到，一个长期低调但拥有全球最大消费电子生态之一的科技巨头，正在快速进入大模型核心战场。

**关键**：两种开篇都不要一上来就说「XX 公司宣布……」。

## 三、结构模板

### 模板 A：产品/融资类

1. **开篇钩子**（2–4 段）：微观场景或痛点，不出现公司名
2. **新闻锚点**（1–2 段）：融资金额+投资方、产品发布、引出公司全称和简称
3. **团队/背景**（可选，1–2 段）：学术+产业交叉背景，履历要具体
4. **技术深度**（2–4 段）：每个模块单独小标题，「做了什么→怎么做到→为什么别人做不到」
5. **市场落地**（1–2 段）：具体行业、已有客户/POC、短期商业路径
6. **收尾展望**（1 段）：回扣开篇主题，留有记忆点的句子

### 模板 B：战略/研究综述类

1. **宏观转折开篇**（2–3 段）：行业拐点 → 引出主角
2. **战略信号点题**（1 段）：「这些工作的背后，都在释放一个明确的信号：……」
3. **核心优势分析**（1–2 段）：与通常玩家的差异化，生态/硬件/数据等独特能力
4. **多项工作串讲**（每项工作 2–4 段）：
   - 简介+论文/项目链接
   - 「传统方案遇到的限制 → 新方案的思路」对比
   - 技术核心（用「简单来说」降维）
   - 这项工作的战略意义
5. **团队/人物收尾**（1–2 段）：组织背景+核心负责人的具体履历和成果数字

## 四、核心写作技法

### 「传统 vs 新」对比结构
解释技术创新时，先描述传统方案的瓶颈，再引出新方案如何突破。

> 传统 Transformer 通常要求大量参数常驻内存……但这些设备往往拥有远大于 RAM 的存储空间（ROM）。于是，三星提出了一个新方向：能否利用 ROM 进一步扩展模型能力？

### 「信号」帧架（战略类文章）
用「释放一个非常明确的信号」统一多项动作的战略含义。

> 这些工作的背后，都在释放一个非常明确的信号：XX 正在全面构建自己的 Foundation Model 体系。

### 「简单来说」降维
技术细节之后，用「简单来说」给出一句白话总结。

> Meki 的核心逻辑……简单来说，RAM 负责实时计算，ROM 负责知识存储，模型进一步突破端侧容量限制。

### 「不仅…还…」能力叠加
描述公司独特优势时用叠加句式，避免平铺罗列。

> 三星不仅拥有模型研发能力，还同时具备覆盖手机、家居、机器人到底层芯片的全球最完整终端设备生态。

### 语言特征

| 特征 | 示例 |
|------|------|
| 用「——」做强调破折 | 触觉路线对软硬件协同要求极高——传感器内部涉及微光学结构… |
| 用「」标记关键概念 | 将视觉从「看见」升级为「感知」 |
| 短句 + 长句交替 | 避免连续长句导致的阅读疲劳 |
| 数据要具体 | 「1.3 万次引用」「万卡集群」，不写「大量」「丰富」 |
| 技术词后跟白话解释 | VLA → 「主要依赖视觉和语言理解环境状态与任务目标」 |
| 小标题有画面感 | 「把接触变成可用数据」优于「传感器技术」 |
| 「某种意义上」用于解读 | 「某种意义上，它探索的是：当 AI 进入物理世界后，如何被评测」 |

## 五、长度参考

| 文章类型 | 目标字数 |
|----------|---------|
| 产品/融资快讯（模板 A） | 800–1200 字 |
| 战略/研究综述（模板 B） | 1200–1800 字 |
| 单篇论文解读（模板 C） | 800–1200 字 |

宁可某些段落略短，不要凑字数写废话。

---

## 六、不要做的事

- 开篇就说「XX 公司于 X 日宣布……」
- 用「可以广泛应用于各种领域」式模糊表述
- 大量使用感叹号或「颠覆」「革命性」等标题党词汇
- 团队背景只写头衔不写具体成果数字
- 技术描述只有形容词没有数据
- 结尾重复正文内容，没有新信息或升华
- 「简单来说」之后还是一堆术语
