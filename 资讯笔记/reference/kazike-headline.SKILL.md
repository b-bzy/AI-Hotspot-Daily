# Kazike Headline Skill

把"卡兹克写完的正文"包装成"卡兹克发出来的成品"。本 skill 不写正文——它只负责正文之外的四样东西：**封面标题**、**正文标题**、**开篇词**、**数字标签**。

## 什么时候用这个 skill

用户已经有正文（或者从你这里刚写出来的卡兹克风格正文），想要：
- 给这篇文章起标题（不论是封面大字、文章题目，还是两者都要）
- 写一段开篇词 / 导语 / 开头铺垫
- 补一组小红书 / 公众号风格的 tag

如果用户只是问"这文章写得好不好"，或者是要写正文，**不要触发**这个 skill。

## 一、整体产出格式

最终产出统一按下面的结构返回，方便用户直接复制使用：

```
【封面标题】
<一行长陈述句，"，" 分隔，句号收尾>

【正文标题】
<一句口语化的钩子，可带【】前缀或！收尾>

【开篇词】
<导语段，3 段以上，固定二段式收尾>

【数字标签】
#<主题相关>  #<主题相关>  ...  #<通用 AI>  #<平台流量>  @科技薯
```

四件套要**风格自洽**：封面标题、正文标题、开篇词的关键词应该可以对得上，能让读者第一眼就知道这篇是讲什么。

## 二、封面标题（题图大字）

### 风格特征

- **长度**：约 15–35 个汉字（单/双行），句号 `。` 收尾
- **句式**：典型为 `[动作/主语描述] + ，+ [价值结论 / 反差判断]`
- **气质**：陈述、自信、稍微夸张但不浮夸；不用感叹号，不用问号

### 句式骨架（任选其一）

1. `实测 [产品 / 模型]，[一句价值判断]。`
2. `[一个反差身份 / 反差对象]，[做成了一件牛逼的事]。`
3. `开源 / 发布 / 公开 [作品]，[让你 / 它能干嘛的 benefit]。`
4. `[做了 X 件事 / 试了一下]，[结论]。`

### 写作要点

- 封面是给"还没决定要不要点开"的人看的，必须**一句话讲清楚价值或反差**
- 喜欢用引号 `「」` 或破折号包专有名词（例：`「洁癖.skill」`）
- 不要用 emoji，不要用感叹号，气质要"克制的炫"
- 涉及型号/产品名时**一定保留原始大小写**（Claude Code、MiMo-V2.5-Pro、GLM-5.1）

### 三个真实样本

| 正文核心 | 封面标题 |
|---|---|
| 实测小米 MiMo-V2.5-Pro，配合 Claude Code 表现出色 | `实测小米MiMo-V2.5-Pro，可能是目前国内最适合Claude Code的新模型。` |
| 一个二本女生用免费 AI 跨考北大的故事 | `一个二本的女生，用免费的AI考上了北大。` |
| 开源自己做的 neat-freak Skill | `开源「洁癖.skill」，让你的Agent越用越聪明。` |

## 三、正文标题（文章题）

### 风格特征

- **长度**：约 10–25 个汉字
- **气质**：比封面标题更"口语化、社交化、流量化"；更愿意用 `！`、`【】`
- **作用**：在公众号列表 / 小红书 feed 流里抓眼球

### 三种典型变体（按正文类型选用）

**A. 实测/测评型** —— 适用于新模型、新工具、新产品的实测
- 骨架：`实测 + [产品名] + ，+ [一句惊叹 / 行业判断]！`
- 例：`实测小米新模型，第一梯队迎来新选手！`

**B. 故事/经验型** —— 适用于人物故事、案例采访、经验分享
- 骨架：`【经验帖】/【访谈】+ [短主语] + ，+ [一句话核心结果]。`
- 例：`【经验帖】她，用AI从二本逆袭到北大。`

**C. 开源/教程型** —— 适用于发布工具、开源项目、写教程
- 骨架：`今天，我[决定]+[动作动词]+「[作品名]」+【附网址】/【附教程】`
- 例：`今天，我决定开源「洁癖.Skill」【附网址】`

### 写作要点

- **能用短句就不要用长句**——正文标题要"快"
- 主语经常缩成代词：`她` / `它` / `我` / `他`
- 喜欢"冒号 + 短结论"或"逗号 + 一句话"的节奏
- 涉及附加资源（链接、教程、Demo）时常带 `【附XX】` 后缀

## 四、开篇词（导语段）

开篇词是卡兹克风格里**最难仿、也最关键**的部分。它要在 250–450 字之间把读者从"刷到"拉到"想看"，并自然地把人引到正文。

### 整体结构（三段式）

```
[起手 hook]  →  [中段铺陈/共鸣/转折]  →  [固定收尾二连]
```

### 起手 hook（三选一）

**起手① 时间锚点**（适合"新东西/新事件"类正文）
- 模板：`昨天凌晨/今天早上/前两天，[谁]+[做了什么大事件]。`
- 例：`昨天凌晨，没有任何预兆的情况下。小米的MiMo-V2.5和MiMo-V2.5-Pro上线了。`

**起手② 故事钩子**（适合"案例/人物/采访"类正文）
- 模板：`我们公司有个 X / 前几天，我朋友 X / 我老婆昨天跟我说...，[一个抓人的小事件]。`
- 例：`我们公司有一个运营的小伙伴，他和我一样，都是二本出来的。前两天的时候，他说他母校的校友群直接炸了...`

**起手③ 直接陈述**（适合"开源/工具/方法论"类正文）
- 模板：`今天没选题了，所以... / 今天，我想... / 这段时间我一直在... / 大家好，我是卡兹克...`
- 例：`今天没选题了，所以想开源一个我自己做的，已经用了快1个多月，迭代了好多版的一个我觉得很有用的Skill。`

### 中段铺陈（5 个常用积木，按需混搭）

1. **行业共鸣 / 现状吐槽**："最近不知道怎么了，各种大模型真的发的一个赛一个一个猛..."
2. **个人偏好背书**："我是十二年的米粉，全家电器都是小米..."
3. **痛点共鸣**："你大概也遇到过：代码迭代了七八轮，文档还是最初那一版..."
4. **触达 / 采访经过**："拜托我们小伙伴去联系...终于得到机会，做了一次专访..."
5. **态度转折 / 超预期**："不过试完之后，有一说一，是有点超出我的预期的..."

### 固定收尾二连（**强制**）

无论起手是哪种，开篇词最后**必须**是这两段，且原话照搬不要改写：

```
[一句话钩子，承接到正文，三选一]
- 老规矩，话不多说，我们正文见！
- 话不多说。我们，开始吧。
- 老规矩，已经在我自己的Skills仓库里面开源，所有人都可以随意使用，地址放在正文里啦，我们正文见！

[然后直接接这一句，原话不变]
如果觉得有用的话，欢迎把它分享给你的朋友们一起看看，这对我们有特别大的帮助！
```

### 语言质感（写到位的细节）

- 多用**短句**和**逗号停顿**，"读起来像在跟朋友讲"
- 大量用语气词：`啊`、`嘛`、`其实`、`说实话`、`有一说一`、`相信我`、`说人话`
- 句号用得多，比起长复合句，卡兹克更喜欢"一句一断"
- 中文里夹专有英文词时，前后**留空格**（"用Claude Code"是错，"用 Claude Code" 是对的，但卡兹克自己也常常不留空格——按正文风格走，正文留就留，正文不留就不留）
- 出现 "。。。" 这种省略号变体很常见，**保留**这种口语化标点

## 五、数字标签（小红书话题 tag）

### 标签构成公式

```
[主题相关 3-5 个] + [通用 AI 2-3 个] + [平台流量 1-2 个] + [固定 IP/@ 1-2 个]
```

### 标签库（按类别）

**🔒 几乎每篇必带（强烈推荐放进去）**
- `@科技薯` —— 小红书科技领域官方号，必带
- `#RED新生代创作大赛` —— 高频流量话题

**👤 个人 IP 类（至少带 1 个）**
- `#数字生命卡兹克`
- `#KapibalaAI`
- `#AIChannel`

**🤖 通用 AI 类（带 2–3 个）**
- `#AI`
- `#AI工具`
- `#人工智能`
- `#大模型`
- `#vibecoding` / `#vibecoding大赏`
- `#AI新手村`

**🎯 主题相关（按正文动态生成 3–5 个）**
- 正文讲 X 模型 → `#X` `#X发布` `#X实测`（如：`#小米MiMo`、`#GLM` `#Claude`）
- 正文讲考研/教育 → `#26考研` `#二本考研` `#考研经验贴` `#北京大学`
- 正文讲开源/工具 → `#开源` `#skill` `#个人开发者`
- 正文讲编程 → `#编程` `#Vibecoding` `#ClaudeCode`

### 排版要点

- 每个 tag 之间用**一个空格**分隔，不换行
- `@` 提及放最后
- 总数控制在 **9–12 个** 之间，多了被限流，少了曝光不够

### 三个真实样本

```
案例1（实测模型类）：
#RED新生代创作大赛 #数字生命卡兹克 #KapibalaAI #AI #AI工具 #小米MiMo #大模型 #人工智能 #vibecoding大赏 #AI新手村 @科技薯

案例2（人物故事类）：
#北京大学 #AI #AI工具 #26考研 #二本考研 #考研经验贴 #AIChannel #RED新生代创作大赛 #AI新手村 @科技薯

案例3（开源工具类）：
#AI #数字生命卡兹克 #RED新生代创作大赛 #AIChannel #AI工具 #开源 #skill #个人开发者 #人工智能 #vibecoding @科技薯
```

## 六、生成流程（执行时按此顺序走）

当用户给出正文（或指向一份正文）后：

1. **识别正文类型**：实测/测评型？故事/采访型？开源/教程型？
2. **抽核心要素**：主体（谁/什么产品）、动作（做了什么）、结果/价值（带来什么）、反差点（如果有）
3. **生成封面标题**：套用第二节句式骨架，强调"价值或反差"
4. **生成正文标题**：根据类型选 A/B/C 变体
5. **生成开篇词**：根据类型选起手 hook，挑 2–3 个中段积木拼接，**必接固定收尾二连**
6. **生成数字标签**：动态主题词 3-5 个 + 通用 AI 2-3 个 + 平台流量（必含 RED新生代创作大赛）+ 个人 IP（至少 1 个）+ `@科技薯`
7. **自检**（见第七节）
8. **按"整体产出格式"统一返回**

---

## 七、自检清单（产出前对照一遍）

- [ ] 封面标题以 `。` 收尾，不超过 35 字，句中含 `，`
- [ ] 正文标题选了 A/B/C 三种之一，气质比封面更"口语 / 社交"
- [ ] 开篇词起手 hook 明确（时间锚点 / 故事钩子 / 直接陈述三选一）
- [ ] 开篇词收尾二连**原话照搬**（"我们正文见！" / "话不多说，我们，开始吧。" + "如果觉得有用的话...特别大的帮助！"）
- [ ] 开篇词全长在 250–450 字
- [ ] 数字标签 9–12 个，含 `@科技薯` 和 `#RED新生代创作大赛`
- [ ] 数字标签里至少有 1 个个人 IP 类（数字生命卡兹克 / KapibalaAI / AIChannel）
- [ ] 四件套关键词风格自洽（封面、正文、开篇都在围绕同一个核心讲故事）
- [ ] **没有**：emoji、过多感叹号（除正文标题）、过度官方腔（"诚邀您"、"敬请期待"）
- [ ] 专有名词大小写正确（Claude Code、MiMo-V2.5-Pro、GLM-5.1、cc-switch）

## 九、常见问题

**Q：用户只让我生成"标题"，要不要把开篇词和标签也给？**
A：不要自作主张。如果用户只点了某一项，就只生成那一项，但可以在末尾加一行 *"需要我顺便配一组开篇词 / 数字标签吗？"* 让用户决定。

**Q：用户的正文风格不是卡兹克，但要求按卡兹克风格包装，怎么办？**
A：照常按本 skill 包装，不必改写正文。但产出后提醒用户：*"正文风格和卡兹克的开篇/标题风格之间会有一点割裂感，你可以让我顺便把正文也调成卡兹克风格。"*

**Q：正文很短（比如只有 200 字），开篇词还要写 300 字吗？**
A：不要。开篇词长度 ≈ 正文长度 / 4，但不少于 150 字。短正文可以省掉中段铺陈，只保留"起手 + 收尾二连"。

**Q：用户给出多个正文，要批量处理？**
A：可以批量。但每篇都要单独走一遍"识别类型 → 抽要素 → 生成 → 自检"流程，不要把所有篇做成同一个模板。
