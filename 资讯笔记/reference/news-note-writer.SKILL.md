# news-note-writer

将一条 AI / 科技新闻扩写为机器之心风格的中文资讯文章。

## 输入格式

你会收到以下一种或多种形式的素材：

1. **早报摘要**：来自 AI Daily Briefing 的结构化摘要，包含标题、「发生了什么」、「关键数据」、「为什么重要」和原文链接
2. **原文正文**：通过 web_fetch 抓取的原始文章内容
3. **话题分类**：`sales`（AI for Sales）/ `voice`（语音相关）/ `ai-general`（AI 综合）

如果只有摘要没有原文，基于摘要写作，并在文末注明「原文来源：[链接]」。

## 第一步：判断文章类型

在写作前，先根据素材判断文章属于哪种类型，选择对应结构：

- **类型 A · 产品/融资类**：单一产品发布、融资公告、功能更新
  → 微观场景开篇 + 新闻锚点 + 技术深度 + 市场落地
- **类型 B · 战略/研究综述类**：公司多篇论文/多项动作、赛道战略布局
  → 宏观行业转折开篇 + 战略信号 + 多项工作串讲 + 人物/团队收尾
- **类型 C · 单篇论文解读**：单篇 arxiv 或研究报告
  → 问题痛点开篇 + 方法核心 + 实验数据 + 行业意义

## 第二步：按风格指南写作

**在写作前先读取** `references/style-guide.md`，按对应模板和语言规范写作。

核心要点（详见风格指南）：

- **开篇不出现公司名**：A 类用物理场景切入，B 类用行业拐点切入
- **数据要具体**：「1.3 万次引用」「成功率提升 90%」，不写「大量」「显著」
- **「简单来说」降维**：每段技术解释后，用一句白话点明核心
- **「传统 vs 新」对比**：解释创新时先写旧方案的瓶颈，再引出新方案
- **小标题有画面感**：「把接触变成可用数据」优于「传感器技术」
- **结尾回扣开篇**，留下有记忆点的句子

## 话题视角调整

根据 `topic_category` 调整写作重心：

- **sales**：强调对销售团队、业务流程、效率提升和 ROI 的影响
- **voice**：关注语音体验、技术突破（情感表达、实时性、多语言）和典型场景
- **ai-general**：平衡技术深度与行业影响，B 类战略文章多属此类

## 输出格式

严格按照以下结构输出，不要省略任何部分：

```
---
date: [YYYY-MM-DD]
source_url: [原文链接]
topic_category: [sales|voice|ai-general]
article_type: [A|B|C]
---

# [主标题]

[正文，800–1800 字，机器之心风格]

---

## 标题备选

1. [选项一：信息量优先]
2. [选项二：场景感/悬念优先]
3. [选项三：数据钩子优先]

## 配图方向

[1–2 句话描述适合配套的图片主题]

## 小红书标签

#标签1 #标签2 #标签3 #标签4 #标签5 #标签6 #标签7
```

## 质量自查

写完后过一遍：
- [ ] 开篇有没有直接出现公司名或「宣布」？（有的话改掉）
- [ ] 正文是否包含至少 2 个具体数字？
- [ ] 技术解释后有没有「简单来说」的白话总结？
- [ ] B 类文章：是否用「信号」帧架统一了多项动作的战略含义？
- [ ] 结尾是否回扣了开篇的核心主题？
