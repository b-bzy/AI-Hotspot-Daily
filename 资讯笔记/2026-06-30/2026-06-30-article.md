# 不用登录门户，一条 WhatsApp 就能下完 B2B 订单：三大商用 Agent 把电商全链路接管了

一个采购经理想给公司补一批货。过去他要做的事是：打开供应商门户，输入账号密码，翻到产品目录，逐项确认库存，再核对自己公司谈下来的那份合同价，最后提交、等待人工客服回邮件确认到货时间。整套流程下来，半小时起步，而且每一步都可能卡住——库存数字是隔夜的，合同价要翻邮件，到货日期客服还得再去问仓库。

现在，他只需要在 WhatsApp 上发一句话。

6 月 29 日，Salesforce 旗下 Agentforce Commerce 的三大核心 Agent 正式宣告全面可用（GA）。这不是又一个挂在网站右下角的聊天机器人，而是三个能真正「办事」的智能体，分别接管了电商的不同环节：Shopper Agent 部署在品牌自有店面，直接回答顾客问题并完成下单；Buyer Agent 通过 WhatsApp 或短信处理 B2B 采购，全程不需要登录任何门户；Merchant Agent 则在后台干活，负责商品目录更新和促销活动管理。

## 关键不在「会聊天」，而在「连得通」

把这三个 Agent 和普通客服机器人区分开的，不是对话能力，而是它们背后的数据连接。

传统的电商聊天机器人，本质上是一个问答接口——它能回答「这件商品多少钱」，但答不了「我这单什么时候能到」，因为它和库存系统、订单系统、客户数据是割裂的。一旦涉及真实交易，它就得把人转接给客服。

Agentforce 的三个 Agent 原生接入了库存、订单管理和客户数据。简单来说，Agent 能直接承诺一个真实的到货日期，能自动调用某个客户的合同专属定价，还能把对话上下文一路带进售后服务环节，中间不需要任何人工转接。对采购经理那个场景而言，这意味着库存是实时的、合同价是自动认的、到货时间是当场给的。

## 销售数据已经站在 Agent 这一边

为什么 Salesforce 选在这个节点把电商 Agent 全面商用？因为 AI 导购正在被真金白银验证。

按 Salesforce 给出的数据，已经用上 AI 导购 Agent 的品牌，销售增速比同行快 59%。更宏观的一个数字是：AI 如今已经影响了全球在线销售额的 20%，折算下来约 2620 亿美元——大致相当于一整个节日购物季的体量。这也解释了发布的时间点：赶在年底购物高峰之前 GA，留给品牌方的正是一个 Q3 调试、Q4 收割的窗口。

而真正可能改变格局的，是商品目录正在被推向主流 AI 入口。Salesforce 同步给出了集成时间表：ChatGPT 商品目录直连集成计划 7 月 GA，Google Search 的 AI Mode 和 Gemini App 则在夏末跟进。换句话说，当越来越多的人开始在 ChatGPT 里直接问「帮我找一双适合越野跑的鞋」时，品牌的商品有没有接进这些入口，会直接决定它出不出现在答案里。值得注意的是，Salesforce 的设计思路是「发现交给第三方助手，结账和数据留在自有渠道」——让 AI 帮你被看见，但不把成交和客户数据拱手让人。

支撑这套「自有渠道留客」逻辑的，还有一个容易被忽略的模块：Agentic Commerce Search。它建立在 Salesforce 此前收购 Cimulate 拿到的技术之上，做的是站内的「意图感知」搜索——不再是传统那种堆关键词、靠规则匹配的笨办法，而是读取顾客在浏览和购买中留下的上下文信号，去主动推出对的商品。简单来说，它想解决的是同一件事的另一半：既然品牌不愿意把成交让给外部助手，那自己的站内搜索就得足够聪明，才接得住从 AI 入口导回来的流量。这个模块同样计划 7 月 GA。

## 给开发者的那一半

面向 C 端的同时，Salesforce 也没落下开发者。随三大 Agent 一起发布的还有一套 Agentic B2C Developer Toolkit，引入了统一的 CLI、MCP Server 和 IDE 扩展，允许开发者用自然语言驱动 AI 编程代理完成组件部署和沙盒创建。配套的 Storefront Next 也已 GA，能自动配置运行时、路由默认域名、创建 GitHub 仓库并交付一套可直接上生产的模板——一句话，把过去搭店面要写的那堆脚手架代码省掉了，让团队把精力放在体验和数据打通上。

## 从「试一个机器人」到「把流程跑起来」

回到那个采购经理。他省下的那半小时背后，是 Salesforce 想讲的一个更大的故事：电商团队的工作重心，正在从「要不要试试 AI 客服」转向「怎么把 Agent 的工作流真正跑顺」。

而一旦进入这个阶段，决定成败的往往不再是模型本身有多聪明，而是更朴素的东西——商品数据干不干净、库存数字准不准、订单状态能不能实时回传。Agent 能不能当场承诺到货日期，取决于它背后那条数据链有没有断点。

还有一道更现实的坎：当 Buyer Agent 可以不登录门户、直接在 WhatsApp 上替企业下单，「这单到底是不是本人下的」就成了绕不开的问题。便利的另一面是权限边界——谁能授权、额度多大、出了错谁负责，这些治理细节 Agentforce 原生接入合同定价和客户数据时能解决一部分，但真正放到企业采购流程里，法务和风控这关恐怕比技术这关更难过。所以这一波 GA 真正的考验，可能不在演示台上，而在每家公司自己的数据底座和审批流里。
