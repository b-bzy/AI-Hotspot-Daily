# 新闻串烧写作规范（形态 B · 供云端周报 routine 使用）

> 提炼自口播写作 skill 的「形态 B 新闻串烧」。一句话定义：**把一段时间几十条动态，硬切成"一行一个信息点"的高密度口播快串——不解说、不算账、只砸事实和数据**，判断只留到收尾一下。

## 整体框架

```
# 标题（点明"串烧版"+覆盖周期）
制作信息（4 行，配音不读）
开场钩子（1 行：时长承诺 + 悬念数）
【聚类 1·主线】最炸的大事，3–4 行，含反转
【聚类 2】… 一行一个信息点，同类成串
…（共 8–12 个聚类）
【彩蛋】1 个轻量反差聚类调剂
【升华收尾】一组反差金句 + 下期预告
```

## 制作信息抬头（4 行，放标题下正文前，配音不读）

- **覆盖周期**：日期范围
- **风格**：硬切快串，无连接词、无解说腔，一行=一个信息点
- **节奏**：5–5.5 字/秒 · 总时长 2.5–2.8 分钟 · 信息行数 · 数据点数。【】为分镜聚类锚点（配音不读）
- **信条**：信息密度第一。能用数据就上数据，能并就并成串。

## 开场钩子（1 行）

时长承诺 + 悬念数，一句进入。例：「三分钟，盘完这一周 AI 圈炸了多少次。」

## 聚类主体（核心）

- 30–40 条分成 **8–12 个【聚类】**，聚类名带能量后缀：`·主线` `·三连` `·井喷`，或硬词命名 `资本巨浪 / 格局 / 人才 / 治理`
- **第一个聚类 = 本周最炸的主线大事**，多给 3–4 行，带反转（"最魔幻的是——…"）
- 聚类内一行=一个信息点，硬切，同类并成串（一组连发多家公司/多个数字）

## 行级铁律（串烧的命门）

1. **一行一个信息点**，句末句号，一行塞 1–3 个硬数字
2. **无连接词**：禁写「另外 / 与此同时 / 值得一提 / 不仅…还 / 此外 / 同时」，直接硬切下一条
3. **无解说腔**：禁写「我替你 / 咱们算笔账 / X 是什么概念呢 / 也就是说」——判断只允许出现在收尾
4. **数据优先于形容**：能上数字绝不用"很大/很多"；能并就并（"A 拿 1.5 亿、B 12.5 亿、C 9.2 亿"）
5. **网感词点到为止**：炸了 / 魔幻 / 痛失 / 火力全开 / 炸锅 / 难产——一个词带过不展开
6. **术语不展开**：生僻术语最多两三个字带过

## 升华收尾（唯一允许出态度的地方）

- 一组**对照/反差洞察**（"一边在被管，一边在狂奔"这种结构）
- 收口金句 + 下期预告：「这，就是本周的 AI。下周，接着炸。」

## 自检清单

- [ ] 制作信息抬头 4 行齐全？
- [ ] 开场 1 行 = 时长承诺 + 悬念数？
- [ ] 8–12 个【聚类】名字有能量？主线打头带反转？
- [ ] 一行一信息点、无连接词、无解说腔？
- [ ] ≥30 信息行 / ≥30 数据点？
