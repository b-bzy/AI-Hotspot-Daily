# 爆款选题评分框架设计文档

> 用途：作为「AI 热点抓取」skill 的固定模块，每天自动跑。输入是当日去重后的候选选题（items.json）+ 四视角编辑打分，输出是一份可直接拿去写稿的「TOP N 爆款选题榜」。
> 定位：这不是给算法看的分数，是给**人**用的决策工具——它替创作者回答三个问题：今天写哪条、用哪个平台/skill 写、怎么写不翻车。

---

## 一、爆款选题评分框架（核心方法论，可复用）

### 1.1 设计原则

一条选题能不能爆，本质是四股力量的乘积，不是加法：**有没有人看（热度）× 看了有没有情绪（张力）× 有没有动机转发（社交货币）× 你有没有独家角度（信息差）**。任何一项接近 0，选题就废。所以框架用「加权求和定基线 + 乘性惩罚砍红海」的混合结构，而不是纯线性加总——纯加总会让「全网都在写的大热点」永远排第一，把差异化蓝海选题全部淹没。

四视角编辑的打分（小红书/公众号/口播/吃瓜）是**证据输入**，不是最终结论。某条选题被几个视角同时看中（breadth）说明它跨平台普适；被单一视角打高分（peak）说明它在某个形态里是尖货。框架要同时奖励这两种情况。

### 1.2 六个评分维度

每个维度 0–10 打分，下面给权重、打分锚点、正反例。

| 维度 | 权重 | 它在回答什么 |
|---|---|---|
| ① 热度势能 Heat | 20% | 这事现在有多少人在看/在讨论 |
| ② 情绪张力 Emotion | 25% | 看完会不会有强情绪（愤怒/震撼/爽/焦虑/共情） |
| ③ 社交货币 Currency | 20% | 读者会不会为了「表达自己」而转发/收藏 |
| ④ 受众共鸣 Resonance | 15% | 跟普通人/目标粉丝的生活决策有多近 |
| ⑤ 角度空间/信息差 Angle | 20% | 你能不能做出别人没有的独家解读/独家清单 |
| ⑥ 竞争红海度 Redsea（负向） | ×惩罚系数 | 是不是全网都在写、你进场已经晚了 |

前五项加权求和得「基础分」（满分 100），第六项作为**乘性惩罚系数**（0.75–1.0）再压一刀。

#### ① 热度势能 Heat（权重 20%）
衡量客观流量信号：HN 分数、掘金/微博阅读、是否上榜、是否多源同时报道。
- **9–10**：全网头条级。HN 1400+ 分 / 微博热搜 / 多平台同时刷屏。例：GPT-5.6 发布（HN 1428 头名）。
- **6–8**：单平台已验证有量。例：撇号追踪（掘金爆文）、1400 Skills 清单（掘金 4852 阅读）。
- **3–5**：垂直圈层有讨论，未破圈。例：蚂蚁灵波世界模型（开发者圈知道，大众无感）。
- **0–2**：几乎无信号，纯冷启动。
- **反例陷阱**：热度高 ≠ 能爆。GPT-5.6 热度满分，但角度空间被全网瓜分，红海惩罚会把它压下来。

#### ② 情绪张力 Emotion（权重 25%，最高权重）
爆款的第一驱动力。判断这条选题能点燃哪种情绪，以及强度。
- **9–10**：多重强情绪叠加。撇号事件 = 愤怒＋民族情绪＋悬疑＋群体焦虑（四燃料）；机器人活体手术 = 震撼＋恐惧＋伦理争议。
- **6–8**：单一强情绪或中等叠加。豆包千问关停 = 背叛感＋焦虑；GPT-5.6 重置额度 = 打脸爽感＋看热闹。
- **3–5**：情绪需要靠标题硬拱。英伟达受害者论、AI ROI 辩论——本身是财经理性题，得靠「反转叙事」才有情绪。
- **0–2**：纯信息、无情绪。常规模型参数更新、KOL 解读转述。
- **判断口诀**：这条能不能让人在评论区**站队开撕**、或者发出一句**「卧槽」**？能，就是 8 分以上。

#### ③ 社交货币 Currency（权重 20%）
读者转发是为了塑造自己的形象。判断它提供哪种「转发动机」：
- **炫耀型**（我懂行）：清单体、横评、深度解读。例：1400 Skills 筛 5 个、三工具横评。
- **站队型**（我表态）：争议、民族情绪、粉黑对线。例：撇号事件、OpenAI 抄袭 Claude。
- **优越型**（我早说过）：打脸、真香、翻车。例：Claude 重置额度、Atlas 8 个月关停。
- **实用型**（我帮你）：迁移方案、避坑指南。例：豆包千问迁移三步走。
- **9–10**：同时具备 2 种以上动机。**6–8**：明确 1 种。**3–5**：动机弱，靠金句撑。**0–2**：转了显得无聊。

#### ④ 受众共鸣 Resonance（权重 15%）
选题离目标粉丝的**真实决策/真实焦虑**有多近。
- **9–10**：直接命中钱包/工作/学习决策。GPT-5.6「我该换订阅吗」、三工具横评「我该买哪个」。
- **6–8**：能共情但不涉及个人决策。豆包关停（用过的人痛）、AI 应用没爆发（每个人都有体感）。
- **3–5**：需要想象力才能共情。机器人手术（离普通人远，但「未来看病」可想象）。
- **0–2**：纯 B 端/纯行业内幕。Meta 自研芯片、英伟达护城河——老板和投资人看，泛用户无感。
- **注意**：受众共鸣要**按账号定位调**。做开发者号，topic 9/11/20 的共鸣分要 +2；做泛科技号则 -2。这是全框架里唯一需要按账号人设手动校准的维度。

#### ⑤ 角度空间/信息差 Angle（权重 20%）
你能不能做出**别人没有的东西**——这是决定「红海里能否杀出来」的关键，也是唯一能靠创作者能力提分的维度。
- **9–10**：有独家清单/独家数据/独家判断/祛魅空间。撇号事件（市场缺一篇讲清技术真相的冷静文）、AI 应用没爆发（需要真判断的常青认知文）。
- **6–8**：有明确差异化切口。GPT-5.6 不写参数写「成本 1/3 背后的技术路径」；马斯克示好挖「400 亿营收命脉」。
- **3–5**：角度已被写烂，只能拼执行。常规发布会复读。
- **0–2**：没有增量，转述即抄袭。
- **核心洞察**：**热度和角度往往成反比**。越热的选题（GPT-5.6）角度越被瓜分；越冷门的反差选题（自动驾驶盯上睡眠）角度空间越大。框架要保护后者。

#### ⑥ 竞争红海度 Redsea（负向乘性惩罚）
不是加分项，是**砍分项**。判断「全网都在写、你进场已晚」的程度，转成惩罚系数：
- 全网头条、当天几百篇同题（GPT-5.6、机器人手术）→ 系数 **0.80–0.85**。
- 有一定竞争但角度可差异化（OpenAI 战略、英伟达受害论）→ 系数 **0.90–0.95**。
- 蓝海/差异化选题，几乎没人认真写（撇号祛魅、自动驾驶盯睡眠、1400 Skills 清单）→ 系数 **1.0**（不罚）。
- **关键逻辑**：红海惩罚是「蓝海保护机制」。它让一条角度独特的中热度选题，最终能排到全网红海大热点前面——这正是这个框架相对「谁热谁第一」的价值所在。

### 1.3 爆款指数计算方式

```
基础分 = Heat×2.0 + Emotion×2.5 + Currency×2.0 + Resonance×1.5 + Angle×2.0
        （六维前五项，满分 100）

爆款指数 = 基础分 × 红海惩罚系数(0.80~1.0)

四视角校准（可选，用四位编辑打分做外部验证）：
  峰值信号 peak = 四视角里的最高分
  广度信号 breadth = 被几个视角选中 / 4 × 100
  若 |爆款指数 - (0.5·四视角均分 + 0.3·peak + 0.2·breadth)| > 12，则人工复核该条
```

四视角均分/峰值/广度是**交叉验证锚**：如果框架算出来的分和四位真人编辑的综合判断差太多，说明维度打分或红海系数拍错了，触发人工复核。日常自动跑时，直接用「爆款指数」排序即可。

**分档解读：**
- **90+**：今日必写头牌，优先级最高，当天出稿。
- **80–89**：强推，值得投入做深。
- **70–79**：可做，看创作者精力和账号定位取舍。
- **60–69**：备选/补位，蹭热度或穿插用。
- **<60**：本日不建议，进选题池留存。

### 1.4 平台形态匹配（同一选题，三平台三种打法）

同一条选题在不同平台是**完全不同的产品**。框架给每条选题标注「最适合平台 + 对应写作 skill」，判断逻辑如下：

| 平台 | 吃什么 | 对应写作 skill | 判断信号 |
|---|---|---|---|
| **小红书图文** | 视觉钩子、清单体、身份认同、封面强对比数字、生活化决策焦虑 | `kazike-tuwen`（卡兹克图文）＋ `file-tree-infographic`/信息图做封面 | 选题能做成「我替你试了 N 个」清单、能做强对比封面、能命中「我该买/该学哪个」→ 小红书 |
| **公众号深度** | 认知增量、趋势判断、把碎片新闻拼成一个洞察、算账、祛魅 | `kazike-tuwen`（长文）／`fenceng-xushi`（分层叙事）／`news-note-writer`（机器之心风资讯） | 选题有独家判断空间、能承载 2000 字、行业人/投资人爱转做谈资 → 公众号 |
| **短视频口播** | 悬念钩子、画面张力、情绪起伏、可视化对比、戏剧化包装 | `keji-koubo-jieshuo`（科技口播，深度解说/新闻串烧两形态）＋ 下游 `koubo-fenjing-shipin` 出画面 | 选题有震撼画面/音频/数字对比、有人物冲突、开头一句能卡住悬念 → 口播 |
| **五级测评（跨平台）** | 给多个模型/工具排序、「夯到拉」框架 | `cong-hang-dao-la`（五级测评） | 选题本质是「给 N 个 AI 产品排个序/打个评价」→ 走五级测评，成稿再套卡兹克图文风 |

**判断一条选题最适合哪里的三步法：**
1. **看情绪载体**：情绪靠画面/音频承载（机器人手术、AI 唱歌）→ 口播；靠认知反转承载（英伟达受害论、AI 没爆发）→ 公众号；靠身份/清单承载（1400 Skills、横评）→ 小红书。
2. **看角度深度**：需要 2000 字才能讲清（战略复盘、算账、祛魅）→ 公众号；一张封面 + 几屏能讲完 → 小红书；30–90 秒能甩完钩子 → 口播。
3. **看是否排序类**：只要本质是「给几个 AI 模型/工具排序打分」→ 无脑上 `cong-hang-dao-la` 五级测评。

一条选题可以标注**主平台 + 次平台**（一鱼多吃）。例如撇号事件：主口播（悬疑侦探叙事）＋ 次公众号（技术祛魅深扒），但**不建议**做小红书（民族情绪 + 事实未核实，翻车风险高）。

### 1.5 「选题卡」输出格式（每条入选选题的标准结构）

每条 TOP N 选题必须输出以下字段，做到「打开就能写」：

```
【排名】#N          【爆款指数】XX（分档：必写/强推/可做/备选）
【选题】一句话说清是什么事
【钩子标题】给 2–3 个可直接用的标题（区分平台语气）
【切入角度】不写什么 + 写什么，一句话锁死差异化
【适配形态】主平台 + 次平台
【适配 skill】具体到哪个写作 skill
【竞争度】红海 / 蓝海（+一句话说明进场时机）
【时效窗口】今天必发 / 3 天内 / 常青（可慢打磨）
【避坑提示】这条最容易翻车/做砸的点
```

## 二、今日爆款选题榜 TOP 10（用框架实跑，2026-07-10）

> 说明：下列排名 = 六维打分 × 红海惩罚，并用四视角编辑打分做交叉验证。特意让 GPT-5.6 这种全网红海大热点接受红海惩罚，把「撇号追踪」「1400 Skills 清单」「自动驾驶盯睡眠」这类差异化蓝海选题顶上来——**这正是框架的价值**。

### #1 撇号追踪中国开发者三个月（争议爆文）
**【爆款指数】93 · 必写头牌**
四视角：小红79 / 公众88 / 口播90 / 吃瓜95 —— 今日唯一四视角全中且吃瓜给出满分级。
- **钩子标题**：
  - 口播：「一个看不见的撇号，让 Claude Code 悄悄盯了中国开发者整整三个月。」
  - 公众号：「一个撇号，'追踪'了中国开发者三个月？我把这事扒到底了。」
  - 小红书（慎用）：「一个看不见的撇号，Claude Code 偷偷追踪中国开发者三个月？到底真假我帮你扒。」
- **切入角度**：**不跟标题党起哄，做技术求真 + 祛魅**。逐层拆解「撇号」的技术真相（是 fingerprint、telemetry 还是地区合规逻辑）、「下毒」说法站不站得住、掘金爆文哪些是事实哪些是情绪，落到「中国开发者用海外 AI 工具的真实合规风险」。
- **适配形态**：主口播（侦探式悬疑叙事）＋ 次公众号（冷静技术深扒）。
- **适配 skill**：`keji-koubo-jieshuo`（口播深度解说）／`kazike-tuwen` 或 `news-note-writer`（公众号祛魅长文）。
- **竞争度**：**蓝海**。掘金已验证争议度，但市场极缺一篇「把技术讲清楚」的冷静文——这就是差异化红利。
- **时效窗口**：3 天内（争议正热，越快越好，但要留出核查时间）。
- **避坑提示**：**最高翻车风险条目**。必须做「指控/争议」定性，不当实锤讲；事实核查要极严谨，否则被反噬。小红书端民族情绪易失控，只做「求证/科普」姿态，不带节奏。

### #2 GPT-5.6 发布（成本 1/3 智能逼近）
**【爆款指数】88 · 必写**
四视角：小红82 / 公众92 / 口播78 / 吃瓜78 —— 热度满分（HN 1428 头名），受红海惩罚仍稳居第二。
- **钩子标题**：
  - 公众号：「GPT-5.6 把价格打到 1/3、智能却逼近 Fable5：OpenAI 这次到底改了什么？」
  - 口播：「同样的智商，价格砍到三分之一——GPT-5.6 这一刀，捅在所有对手命门上。」
  - 小红书：「GPT-5.6 来了！同样的活儿只花 1/3 的钱，我的 ChatGPT 会员要退了吗？」
- **切入角度**：**不做发布会复读机**。公众号深挖「成本降 2/3 智能不降」的技术路径（蒸馏/稀疏化/新架构）＋ 对定价体系的冲击；小红书换成「省钱/我该不该换订阅」的钱包视角；口播做「价格屠夫/刺客登场」的戏剧化。
- **适配形态**：主公众号（技术+定价深度）＋ 次小红书（钱包决策）。
- **适配 skill**：`kazike-tuwen`（公众号/小红书）／`keji-koubo-jieshuo`（口播）。
- **竞争度**：**红海**（红海系数 0.85）。全网当天几百篇同题，纯参数罗列必被淹没，只有靠角度活。
- **时效窗口**：今天必发（时效性极强，晚一天就过气）。
- **避坑提示**：避开 ARC-AGI-3、SOTA 等术语堆砌（尤其小红书）；纯发布会流水账会掉到及格线，必须有独家切口才值得进场。

### #3 宇树 G1 机器人活体动物外科手术（全球首例）
**【爆款指数】86 · 必写**
四视角：小红74 / 公众84 / 口播94 —— 口播视角天生爆款，画面张力拉满。
- **钩子标题**：
  - 口播：「一台机器人，第一次把刀切进了活的身体里——操控它的人，在几十公里外。」
  - 公众号：「全球首例：人形机器人做完了外科手术——具身智能的'iPhone 时刻'来了吗？」
  - 小红书：「全球首例！机器人给活体动物做了外科手术，未来看病是机器人主刀？」
- **切入角度**：口播甩「全球首例」+ 卡悬念 + 血腥张力画面 + 「延迟 0.1 秒会怎样」的惊悚设问；公众号必须**客观界定能力边界**（是「远程操控」不是「自主手术」），讲清为什么依然是里程碑（精度/时延/稳定性）。
- **适配形态**：主口播（强视觉震撼）＋ 次公众号（能力边界深度）。
- **适配 skill**：`keji-koubo-jieshuo` + `koubo-fenjing-shipin`（出画面）／`kazike-tuwen`（公众号）。
- **竞争度**：**红海偏中**（红海系数 0.88）。震撼题材当天会有很多人做，但好画面 + 客观界定能拉开差距。
- **时效窗口**：今天必发（新闻属性强）。
- **避坑提示**：别被「AI 做手术」的耸动标题带偏，专业读者会反感；必须强调「远程操控 ≠ 自主」，否则被打脸。

### #4 GPT-5.6 一发布 Claude 就重置 Fable5 额度（吃瓜调侃）

## 三、如何集成进每日流水线

把「爆款选题排序」做成 `ai-hotspot-daily` skill 的一个**固定后置模块**，在热点抓取和去重之后自动跑。

### 3.1 数据流

```
① 抓取去重（现有）  →  items.json（当日候选选题，已按事件去重）
② 四视角打分（现有）→  四位编辑各自输出 picks（lens + score + hook + angle + reason）
③【新增】爆款排序   →  读 items.json + 四视角 picks，按六维框架打分
④ 输出              →  TOP N 选题卡（markdown），放日报末尾 + 单独存一份
```

### 3.2 可操作步骤

**Step 1 · 定义模块入口。** 在 skill 目录下新增 `scoring/` 子模块，包含：
- `rubric.md`：本文档「一、评分框架」的固化版（六维 + 权重 + 打分锚点 + 红海系数表），作为打分时的固定 prompt 依据。
- `score_topics.py`（或让 Claude 直接按 rubric 打分）：输入 `items.json` + 四视角 `picks.json`，输出 `ranked_topics.json`。

**Step 2 · 打分逻辑（每条选题跑一遍）：**
1. 读入该选题的四视角分数（作为 Heat/Emotion/Currency 的证据锚）。
2. 按 rubric 对六维各打 0–10（Emotion、Angle 权重最高，重点判断）。
3. 计算 `基础分 = Heat×2 + Emotion×2.5 + Currency×2 + Resonance×1.5 + Angle×2`。
4. 判定红海系数（0.80–1.0），得 `爆款指数 = 基础分 × 系数`。
5. 交叉验证：算 `0.5·四视角均分 + 0.3·peak + 0.2·breadth`，与爆款指数偏差 >12 则标记 `⚠️需复核`。

**Step 3 · 平台/skill 匹配。** 对每条选题按 §1.4 三步法自动标注「主平台 + 次平台 + 适配 skill」，并按账号定位调受众共鸣分（在 skill 配置里存一个 `account_profile: 开发者号/泛科技号`，命中开发者向选题时 Resonance ±2）。

**Step 4 · 生成选题卡。** 按 §1.5 格式渲染 TOP N（建议 N=10），每条含：排名、爆款指数、钩子标题×2–3、切入角度、适配形态、适配 skill、竞争度、时效窗口、避坑提示。

**Step 5 · 输出落位：**
- 追加到当日 AI 热点日报**末尾**，作为「今日选题决策建议」板块。
- 同时单独存一份 `ranked_topics_YYYY-MM-DD.md`，方便创作者直接拎去写稿。
- （可选）把 TOP 3 的钩子标题 + 适配 skill 做成一句话推送，早上直接看到「今天写这条、用这个 skill」。

**Step 6 · 反馈闭环（让框架越跑越准）。** 每天实际发文后，回填真实数据（阅读/点赞/转发）到 `feedback.jsonl`。定期比对「爆款指数预测 vs 实际表现」，校准六维权重和红海系数——尤其校准 Emotion 和 Angle 这两个高权重、最主观的维度。这是把框架从「一次性设计」变成「自进化模块」的关键。

### 3.3 与下游写作 skill 的衔接

选题卡里的「适配 skill」字段是**可直接触发的**：创作者选定一条后，把选题卡的「切入角度 + 钩子标题」喂给对应写作 skill（`kazike-tuwen`/`keji-koubo-jieshuo`/`cong-hang-dao-la`/`fenceng-xushi`/`news-note-writer`）即可开写。整条链路做到：**抓热点 → 排选题 → 出选题卡 → 一键转写作 skill 成稿**，全程不需要人重新想「今天写什么、怎么写」。
