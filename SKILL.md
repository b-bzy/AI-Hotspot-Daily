# AI 热点抓取 · 日报流水线（v1.3，25 源）

**一句话**：跑一次脚本抓全部源 → Claude 读 items.json 整理成分类日报 → 落盘 `待审核/YYYY-MM-DD/`。只收集，不打分、不选题、不写文、**绝不自动发布**。

工作区：`/Users/jacob/Desktop/账号运营的内容/AI热点抓取/`
运行配置：`配置.json`（源开关/时间窗/关键词表，改配置不改本文档）
源清单与坑：`reference/sources.md`

## 第一步：幂等检查

```bash
TZ=Asia/Shanghai date +%F   # 用这个日期，避免 UTC 错位
```

若 `待审核/YYYY-MM-DD/AI热点日报.md` 已存在 → 在 chat 报一行「今日已产出，跳过」并退出，**不要覆盖**。
例外：用户明确说「重跑/再抓一次」→ 带 `--force` 重抓并覆盖。

## 第二步：跑抓取脚本

```bash
cd "/Users/jacob/Desktop/账号运营的内容/AI热点抓取"
python3 scripts/fetch_all.py
```

- 脚本自己处理：25 个源抓取、单源失败不阻塞、中文热搜 AI 关键词过滤、跨源 URL 去重、摘要截断
- 产物：`待审核/YYYY-MM-DD/items.json`（全部条目）+ `fetch_report.json`（各源成败）
- 脚本 exit 2 = 超 60% 源失败 → 终止流程，把 fetch_report 里的错误写进报告，等人工处理。此时残缺数据在 `items.failed.json`（不写 items.json，所以重跑不会被幂等挡住）
- **不要重试失败的源**，失败信息如实进日报尾部

## 第三步：Claude 整理日报

**只读 `items.json`**（绝不去读任何原始网页/RSS/HTML），写 `待审核/YYYY-MM-DD/AI热点日报.md`：

```markdown
# AI 热点日报 · YYYY-MM-DD（前 24h，共 N 条，M/25 源正常）

## 🔥 今日焦点（跨源同报的大事，0-5 条）
## 🚀 模型与产品发布（大厂官方博客 vendor_blog 的发布优先归这里，注明「官方」）
## 🏢 公司与行业动态
## 🔓 开源项目与模型（GitHub + HF trending）
## 📄 研究论文（HF Daily Papers）
## 💬 KOL 与社区热议（smol.ai 综述要点 + 官方号推文(zpravobot) + AI博主发言(bluesky) + HN 高分讨论）
## 🇨🇳 中文圈热点（微博/知乎/B站/垂直媒体）

## ⚠️ 本次异常源：xxx（错误一句话）；无异常则写"全部正常"
```

整理规则：
1. **条目格式**：`- **标题** — 一句话摘要 | 热度原值 | [源](url)`，一行一条
2. **合并同一事件**：明显是同一件事的多源条目合并为一条，标注「N 源同报」并把链接都挂上；跨中英文也要合并。**多源同报的自动进「今日焦点」**
3. **踢明显漏网**：关键词误放进来的非 AI 条目直接丢弃（比如带"机器人"的娱乐新闻）
4. **smol.ai 的处理**：它的 summary 是前一天 X/Reddit/Discord 的长综述，从中提炼 3-6 个要点作为独立条目放进「KOL 与社区热议」，注明出自 smol.ai
5. **排序**：板块内按热度原值/多源命中降序；宁多勿删，除第 3 条外不做筛选——这是收集不是策展
6. **允许平静**：真没大事，焦点板块写「今日平静」即可，不硬凑

## 第三步半（可选）：推送 Telegram

```bash
python3 scripts/send_telegram.py
```

- 工作区根目录存在 `telegram配置.json`（bot_token + chat_id）时才真正发送：焦点板块作为消息 + 完整日报 .md 作为附件
- 未配置/占位符状态脚本自动 SKIP（exit 0），流水线无条件调用即可
- 发送失败不阻塞流程，如实写进汇报；token 文件已 gitignore，**绝不能提交**

## 第四步：落盘 + 汇报

1. 确认 `待审核/YYYY-MM-DD/` 下有：`items.json`、`fetch_report.json`、`AI热点日报.md`
2. 在 chat 里输出：
   - 统计行：总条数、各板块条数、异常源
   - 「今日焦点」板块全文
   - 其余板块各贴前 3 条预览
   - 日报文件路径

## 绝对不要做

- 不要自动发布到任何平台；不要动 `待审核/` 里往日的产物
- 不要让 Claude 直接抓/读原始网页替代脚本（烧额度）
- 单源失败不重试、不深究，如实记录即可
- 不要修改 `配置.json` 里用户调过的参数

## 失败处理

任一步骤卡死 → 已有产物原样留在 `待审核/YYYY-MM-DD/`，chat 里说明卡在哪一步、错误原文、需要 jacob 做什么，然后停。
