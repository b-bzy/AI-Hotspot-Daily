# 今日 GitHub AI 热点：智能体应用扎堆上榜

> 今日 GitHub Trending 日榜上，AI 相关项目集中在应用层：LLM 应用合集、官方 API 教程、垂直领域的智能体系统各占一席。三者路径不同，但都在探索"多个模型或角色协作完成复杂任务"这条思路。

---

## 1. awesome-llm-apps — 100+ 个可直接运行的 AI Agent 与 RAG 应用合集

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 118.7k | 408 | Python | Apache-2.0 |

这个仓库解决的问题是"学 AI 应用开发时找不到能直接跑起来的例子"。它收录了超过 100 个 LLM 应用模板，覆盖从入门级单智能体到多智能体协作系统，每个模板都能通过几条命令本地运行，而不是简单的链接收藏。

和很多"awesome list"不同，这里的模板由作者本人开发并测试过，不依赖单一模型厂商，同时支持 Claude、Gemini、OpenAI、xAI、Qwen 和 Llama。仓库分成 15 个类别，包括语音智能体、游戏对战智能体、MCP 智能体和 RAG 系统等，覆盖面较广。

适合正在学习智能体开发、想要一个可复制模板起步的开发者，也适合想快速验证某个 RAG 或多智能体方案是否可行的团队。

🔗 项目地址：https://github.com/Shubhamsaboo/awesome-llm-apps

---

## 2. claude-cookbooks — Anthropic 官方出品的 Claude API 使用教程合集

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 48.5k | 459 | Jupyter Notebook | MIT |

这是 Anthropic 官方维护的 Claude API 实践手册，面向想把 Claude 集成进自己产品的开发者。内容以 Notebook 形式组织，涵盖分类、检索增强生成、摘要等基础能力，以及工具调用、多模态、PDF 解析等进阶用法，代码可以直接复制使用。

和普通 API 文档不同，这个仓库更偏"配方"而非"参考手册"——比如用 Haiku 做 Opus 的子智能体、把 Claude 接入 Pinecone 向量数据库、用 Claude 搭配 Stable Diffusion 生成图片，都是具体到可运行代码的案例，而不是抽象的接口说明。

适合刚拿到 Claude API key、不知道从哪里下手的开发者，也适合想找现成代码片段直接用的团队。

🔗 项目地址：https://github.com/anthropics/claude-cookbooks

## 3. ai-hedge-fund — 用 19 个投资风格智能体模拟对冲基金决策的教学项目

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 61.5k | 115 | Python | MIT |

这个项目用多智能体模拟对冲基金的决策流程：19 个智能体里既有模仿沃伦·巴菲特、查理·芒格、凯茜·伍德等知名投资人风格的分析智能体，也有专门做估值、情绪分析、基本面和技术面研究的智能体，最后由投资组合管理和风险管理智能体汇总给出结论。

项目明确写明这是"教学性质的概念验证"，不用于真实交易或投资建议。它支持命令行和网页两种使用方式，可以对接 OpenAI、Anthropic、DeepSeek、Ollama 等多家模型厂商，并带回测功能，可以指定股票代码和时间区间跑历史数据。

适合想研究多智能体协作模式的开发者，或者对"用 LLM 模拟专家决策"这类应用形态感兴趣的人，不建议直接用于实盘。

🔗 项目地址：https://github.com/virattt/ai-hedge-fund
