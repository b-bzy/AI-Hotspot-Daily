# 今日 GitHub AI 热点：LLM 正被装进交易、决策、语音三个垂直场景

> 今天日榜上排名靠前的 AI 项目，方向各不相同，但有一个共同点：都是把大模型能力封装进某个具体场景，而不是再做一个通用框架。这三个分别落在量化交易、复杂决策和本地语音转写，下面逐个看。

---

## 1. Vibe-Trading — 给 Agent 接上交易能力的个人投研框架

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 15.2k | 839 | Python | MIT |

Vibe-Trading 由香港大学数据智能实验室（HKUDS）维护，定位是一个"个人交易 Agent"框架：用一条命令给 Agent 接上行情获取、财报检索、回测和投研分析的能力，让大模型能围绕一只标的做结构化的研究，而不是只回答泛泛的问题。项目用 FastAPI 做后端、React 19 做前端，已发布到 PyPI（`vibe-trading-ai`），并提供 API 与 MCP 接口。

值得注意的是它在"下单"这件事上的工程取舍。根据 6 月 29 日的更新记录，项目对实盘操作设置了多道闸门：mandate gate（授权门）、kill switch（急停开关）和完整审计日志，新加入的 Trading 212 连接器目前只开放只读的账户、持仓、订单和历史数据，`place_order` / `cancel_order` 在缺少明确的模拟盘与实盘边界前会直接拒绝执行。换句话说，它把"研究"和"真实下单"刻意分开了。

适合想用大模型搭建自有投研流程、且看重风控边界的开发者参考。需要提醒的是，投资决策仍需自行判断，工具本身不构成任何投资建议。

🔗 项目地址：https://github.com/HKUDS/Vibe-Trading

## 2. FluidVoice — 在 Mac 上完全本地运行的语音转写工具

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 4.4k | 830 | Swift | GPL-3.0 |

FluidVoice 解决的问题很具体：在 macOS 上做语音听写，且全程在本机完成，不把音频传到云端。它支持多种语音识别模型，包括 NVIDIA 的 Parakeet 系列、Nemotron Speech 3.5、Cohere、Apple Speech 以及 Whisper，用户可以按设备和语言选择引擎。安装方式是 `brew install --cask fluidvoice`。

和依赖云端 API 的听写工具相比，它的差异点在于"本地优先"。1.6.0 版本重写了 Parakeet 的实现，目标是把说话到屏幕上出字的延迟降到接近零；同时引入了名为 Fluid Intelligence 的本地 AI 层，用于在设备上做智能格式化、上下文相关的大小写处理等后处理，不需要 API key、数据不离开本机。需要说明的是，应用本体按 GPLv3 开源，而 Fluid Intelligence 这一增强层目前由作者单独闭源维护。

适合对数据隐私敏感、或希望离线使用听写的 Mac 用户。

🔗 项目地址：https://github.com/altic-dev/FluidVoice

## 3. Council of High Intelligence — 让多个模型分角色辩论再给结论

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 2.0k | 331 | Shell | MIT |

这个项目的思路是用"结构化的分歧"代替单个模型的一锤定音。它内置 18 个带不同思维传统的 AI 角色（如亚里士多德、苏格拉底、费曼、卡尼曼、Torvalds 等），通过一条 `/council` 命令，让它们围绕一个难题分多轮讨论，最后综合出结论。角色会被分配到 Claude、OpenAI、Gemini、Ollama 等不同供应商，以保证推理路径真正有差异，而不是同一个模型换层皮。

机制上有几处设计针对"附和"问题：Problem Restate Gate 要求每个成员先各自复述问题再开始分析，如果几个成员对问题的理解就不一致，说明问题本身没问清楚；此外还有异议配额、新颖性门槛和反事实提示，当超过七成成员过早达成一致时，会强制两名成员去为对立观点辩护。最终结论会先列出"尚未解决的问题"和"建议的下一步"，而不是先给一个听上去很笃定的共识。它以 Claude Code / Codex 的 skill 形式安装，并提供 `--quick`、`--duo` 等不同规模的讨论模式。

适合需要对重要决策做多角度推敲、且手上有多家模型 API 的开发者。

🔗 项目地址：https://github.com/0xNyk/council-of-high-intelligence

## 今日观察

今天上榜的三个 AI 项目方向各异，但都不是再造一个通用 Agent 框架，而是把大模型能力收敛到一个明确场景里：交易投研、决策推敲、本地语音。一个共同的细节是它们都在认真处理"边界"——Vibe-Trading 把下单挡在闸门后，FluidVoice 强调数据不出本机，Council 则刻意让结论先暴露不确定性。从通用能力转向场景化与可控性，是值得关注的一个方向。
