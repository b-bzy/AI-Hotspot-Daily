# 今日 GitHub AI 热点：金融预测、代码审查、CAD 设计三条 Agent 路线同时上榜

> 今天的 GitHub Trending 日榜上，AI 相关项目没有集中在聊天机器人或通用 Agent 框架，而是分散在金融时序预测、代码审查、CAD/硬件设计三个具体场景。这三个项目分别用不同方式回答同一个问题：怎么把大模型的能力收窄到一个垂直任务上，做到比通用方案更好用。

---

## 1. Kronos — 面向金融 K 线数据的开源基础模型

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 33.1k | 401 | Python | MIT |

Kronos 要解决的问题很具体：K 线（开盘价、最高价、最低价、收盘价、成交量）时间序列预测过去大多要为每个品种单独训练模型，Kronos 的目标是用一个统一的基础模型覆盖不同资产、不同周期的预测任务。据其 README，模型训练数据来自 45 个以上的全球交易所，官方称其为首个开源的金融 K 线基础模型。

技术路线上，Kronos 采用两阶段设计：先用一个专用 tokenizer 把连续的多维 K 线数据量化成分层的离散 token，再用一个仅解码器（decoder-only）的自回归 Transformer 在这些 token 序列上做预训练。项目在 Hugging Face 上开源了三个规格：Kronos-mini（410 万参数）、Kronos-small（2470 万参数）、Kronos-base（1.023 亿参数），预测时支持通过温度和 top-p 采样控制输出的概率分布。该工作已被 AAAI 2026 接收。

适合做量化研究、需要跨品种做时序预测、又不想为每个资产单独训练模型的开发者参考。项目提供了一个实时演示页面，展示对 BTC/USDT 未来 24 小时走势的预测。

🔗 项目地址：https://github.com/shiyu-coder/Kronos

## 2. Open Code Review — 阿里内部代码审查工具的开源版本

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 11.7k | 180 | Go | Apache-2.0 |

Open Code Review（CLI 命令 `ocr`）最初是阿里巴巴集团内部的 AI 代码审查工具，据项目介绍已在生产环境运行两年，服务过数万名开发者，识别出数百万个代码缺陷，验证充分后才开源。它解决的是通用 Agent 做代码审查时常见的问题：改动文件多了容易漏审、报告的问题行号和实际位置对不上。

它的做法是把"确定性工程"和"Agent"分开使用：文件筛选、相关文件打包（比如把中英文文案文件打包成一个审查单元并各自用独立上下文的子 Agent 处理）、审查规则匹配这些步骤交给工程逻辑保证稳定，Agent 只负责需要动态判断的部分，比如结合代码上下文生成审查意见。项目给出了一份基准测试：基于 50 个开源仓库、200 个真实 PR、10 种编程语言，由 80 多名资深工程师交叉标注出 1505 条问题作为标准答案。测试显示，在相同底层模型下，相比通用 Agent（如 Claude Code）做审查，它的 Precision 和 F1 更高，消耗的 token 约为通用 Agent 的九分之一，代价是召回率（Recall）更低。

适合已经在用 CI 流程、想把代码审查自动化接入 GitHub Actions、GitLab、Bitbucket、Gerrit 等场景的团队，支持配置 OpenAI 或 Anthropic 兼容的模型接口。

🔗 项目地址：https://github.com/alibaba/open-code-review

## 3. text-to-cad — 面向 CAD、机器人与硬件设计的 Agent 技能库

| 总 Star | 24h 新增 | 主语言 | 协议 |
|---|---|---|---|
| 10.1k | 230 | JavaScript | MIT |

这个项目解决的是"让 Agent 直接生成能用的 CAD 文件"这件事。它不是一个单独的生成模型，而是一套可以安装进 Claude Code、Codex 等编码 Agent 的技能库，一共包含 11 个技能：根据文字或图片描述生成 CAD 模型并导出 STEP/STL/3MF/GLB 格式、本地浏览器预览、生成机器人描述文件（URDF/SRDF）、仿真世界文件（SDF）、2D 图纸（DXF）、G-code 切片，以及在 SendCutSend 之类的加工平台上传前做文件校验。

技术上的差异点在于，它把"CAD 生成"拆成了一串具体的、可验证的子任务，而不是让一个模型端到端输出几何体。项目给出的基准测试用可复现的提示词（比如"生成一个居中的 100×60×20 毫米长方体，打四个 8 毫米通孔，顶部外缘倒 2 毫米角"）配上输出结果的动图，方便核对生成结果是否符合要求。其中的 step.parts 技能还能检索螺丝、轴承、电机、连接件等现成的标准件 STEP 模型，避免每次都从零建模。

适合做硬件原型、机器人结构设计，希望用编码 Agent 直接产出可加工文件、而不是打开传统 CAD 软件手动建模的工程师和 Maker。

🔗 项目地址：https://github.com/earthtojake/text-to-cad

## 今日观察

今天上榜的三个项目有一个共同点：都没有停留在"通用 Agent 能力"上，而是把 Agent 或模型收窄到一个具体垂直任务——预测金融时间序列、审查代码 diff、生成 CAD 文件——并各自配了一套针对该任务的工程约束（tokenizer、确定性流水线、可复现基准测试）。相比"什么都能干"的通用框架，这三个项目更像是在回答"这一件事怎么干得更准"。
