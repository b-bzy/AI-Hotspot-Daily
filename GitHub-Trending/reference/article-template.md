## 几条容易忘的细节

1. **数据栏一定要写完整**：4 列都填，没有的字段写 `—` 而不是留空
2. **协议字段**：用 SPDX 标识（MIT / Apache-2.0 / GPL-3.0），不要写"开源协议"这种空话
3. **项目名大小写保持原样**：Ollama 不是 ollama，llama.cpp 不是 LLAMA.cpp
4. **Tags 数量**：5 个，1 个大流量（#GitHub热点 或 #AI开源）+ 3–4 个具体技术词
5. **如果是补位的非 AI 项目**：在它的"一句话定位"里照实写定位即可，不需要标注"非 AI"

## 反例（不要这样写）

❌ 反例 1（夸张）：
> # 炸裂！3 个 GitHub 神级项目改写 AI 未来！
> 今天 GitHub 上有三个王炸级别的项目登顶...

❌ 反例 2（功能清单）：
> ## 1. Ollama
> 它支持 GGUF 格式、支持流式输出、支持多平台、支持量化、支持 API 调用...

❌ 反例 3（编造数据）：
> 据测试，它比同类项目快 50%（README 里其实没说）

❌ 反例 4（替项目站台）：
> 强烈建议每个 AI 开发者都装一下。

## 正例（一段示范）

> ## 1. Ollama — 在本地跑大语言模型的命令行工具
>
> | 总 Star | 24h 新增 | 主语言 | 协议 |
> |---|---|---|---|
> | 92k | 380 | Go | MIT |
>
> Ollama 解决的问题很具体：让普通开发者在一台 MacBook 或 Linux 工作站上把 Llama、Mistral、Qwen 这些开源模型跑起来，不用配置 CUDA、不用懂 PyTorch。一行 `ollama run llama3` 就能进入对话。
>
> 和 llama.cpp 这种偏底层的工具相比，Ollama 把模型下载、格式转换、显存管理这些步骤封装到了一个守护进程里，并暴露 OpenAI 兼容的 HTTP API。这意味着原本接 OpenAI 的应用，把 base_url 改成 `localhost:11434` 就能直接换成本地模型，不用改业务代码。
>
> 适合做本地 LLM 试验、对数据隐私敏感的场景、以及不想付 token 费的小项目。
>
> 🔗 项目地址：https://github.com/ollama/ollama
