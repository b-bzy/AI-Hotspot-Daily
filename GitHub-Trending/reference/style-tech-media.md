# 正经科技媒体风格指南

> 适用对象：每天产出的 GitHub AI Trending 介绍文章。
> 风格参考：InfoQ / 极客公园 / 机器之心 / The Decoder。
> 不参考：卡兹克唠嗑体、小红书少女体、震惊体公众号。

---

## 核心原则

1. **客观陈述优先于情绪渲染**：写"它把推理速度从 X tok/s 提到 Y tok/s"，不写"它把推理速度搞爆了！！"
2. **数字优先于形容词**：能给数字就给数字。"star 多"不如"24 小时新增 3.2k star"
3. **用户故事优先于功能清单**：写"开发者用它把本地 LLM 跑在 8GB 显存的笔记本上"，不写"它支持量化、支持 GGUF、支持流式输出"
4. **承认不知道好过编造**：README 里没说的就不写，不要瞎补背景
5. **保持中立**：不替项目站台，也不踩同类竞品

---

## 禁用词清单（**严格禁止**）

这些词只要出现一次，文章就废了重写：

| 类别 | 禁用词 |
|---|---|
| 夸张感叹 | 炸裂、王炸、震撼、爆炸、爆裂、绝杀、屠榜、统治、横扫 |
| 价值评判 | 颠覆、革命、改变世界、改写历史、定义未来 |
| 网络流行语 | yyds、绝绝子、神器、太牛了、屌爆、卷王、内卷死了、爆款 |
| 营销话术 | 不容错过、必看、错过后悔、限时、千万别错过 |
| 主观吹捧 | 完美、无可挑剔、史诗级、教科书级、堪比、媲美 |

### 替代方案

- ❌ "这个项目炸裂了" → ✅ "这个项目过去 24 小时新增 4.1k star"
- ❌ "颠覆了 LLM 推理" → ✅ "在常见硬件上把推理速度提高了 2–3 倍"
- ❌ "神器，必看！" → ✅ "适合需要 X 场景的开发者参考"
- ❌ "yyds" → ✅ （删掉这个词）
- ❌ "完美解决了 X" → ✅ "在 X 场景下提供了一个可行方案"

---

## 推荐用词

- 中性动词：发布、迭代、推出、开源、上线、维护、贡献
- 比较：相比 X 提高了 Y%、与同类项目 X 的差异在于、对比测试显示
- 限定：在 X 场景下、在 Y 条件下、对于 Z 用户来说
- 评价：值得关注、可用于、适合 X 类用户、社区反响积极、生态尚未成熟

---

## 长度与节奏

- 文章总长：800–1500 字
- 每个项目：200–400 字
- 段落：每段 2–4 句，避免单句成段或一段 200 字
- 句子：以短句为主，长难句不超过 30 字
- emoji：**全文不用 emoji**（这点和小红书风格不同），唯一例外是项目链接前的 🔗

---

## 数字使用规范

| 数据 | 写法 |
|---|---|
| Star 数 | 整数（"3,200 star" 或 "3.2k star"，统一用 k/m 简写） |
| 今日新增 | "24 小时新增 X" |
| 项目年龄 | 用"创建于 YYYY 年"或"X 个月前开源"，不用"很早"/"很新" |
| 性能提升 | 必须带具体数字，没数据就别说"性能强" |
| 用户规模 | 有 readme 里写的具体数字才写，没有就别写 |

---

## 章节标题规范

- 用项目名 + 一句话定位，不用形容词
  - ✅ `## 1. ollama — 在本地跑 LLM 的命令行工具`
  - ❌ `## 1. ollama — 让本地 LLM 变得无比简单！`
- 章节标题不带感叹号、不带 emoji
- 项目名保持原始大小写（`Ollama` / `LangChain` / `llama.cpp` 各自原样）

---

## 不夸大的具体标准

| 总 star 数 | 允许的描述 |
|---|---|
| < 1k | "新项目"、"刚起步"、"早期阶段" |
| 1k – 10k | "已有一定关注度"、"在 X 圈子里被讨论" |
| 10k – 50k | "已有规模"、"被多个团队使用" |
| 50k – 100k | "明星项目"、"行业内有影响力" |
| > 100k | "头部开源项目"、"事实上的 X 标准之一" |

不要把 5k star 的项目写成"明星项目"。

---

## 立场约束

- 不替项目站台："我推荐你用它" → 改成 "适合 X 场景"
- 不踩竞品："比 Y 强多了" → 改成 "和 Y 的差异在于"
- 不预测未来："会成为 X 标准" → 改成 "目前在 X 方向有探索"
- 不替开发者代言："开发者一定爱用" → 删掉

---

## 校对清单（写完自查）

- [ ] 全文有没有禁用词？
- [ ] 每个数字都来自 metadata 或 README，没有编造？
- [ ] 项目定位是不是"问题导向"（解决什么）而非"功能清单"？
- [ ] 有没有不该有的 emoji？
- [ ] 项目链接是不是用 🔗 引导？
- [ ] 总字数在 800–1500？
- [ ] 没有把小项目吹成明星项目？
