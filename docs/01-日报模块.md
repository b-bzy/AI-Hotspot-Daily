# 模块文档 · ①AI 热点日报(+ 爆款选题榜)

> 平台的创始模块。每天从 25 个中英文源抓取过去 24h 的 AI 热点,整理成分类日报 + 给自媒体选题用的爆款选题榜。

## 每日时间线(GST)

```
07:43  fetch-daily Action(GitHub 虚拟机,全网通)跑 scripts/fetch_all.py
       → 待审核/日期/items.json + fetch_report.json 提交入仓
08:17  云端 routine(trig_014n8LXUnr6Tg8ArfemnjiV4)只读 items.json
       → 写 AI热点日报.md(8板块) + 爆款选题榜.md(TOP10选题卡) → 推 main
08:1x  telegram-push Action → 日报焦点消息+全文附件、选题榜 发到 Hotspot 机器人
08:09  (本地兜底,Mac开机才有) 桌面调度器任务 ai-hotspot-daily
```

## 文件去向

| 文件 | 说明 |
|---|---|
| `SKILL.md`(根目录) | 完整工作流:幂等→抓取→整理规则(8板块/多源合并/宁多勿删)→选题榜→推送 |
| `配置.json`(根目录) | 25 源开关、hn_queries、中英文关键词表、zpravobot/bluesky 账号清单 |
| `reference/sources.md` | 25 源端点+坑+死亡名单+扩展候选(全部实测记录) |
| `reference/爆款选题评分框架.md` | 选题榜的六维评分模型(热度/情绪/社交货币/共鸣/角度 × 红海惩罚),**周报也用它** |
| `scripts/fetch_all.py` | 零依赖抓取脚本(纯标准库);失败不写 items.json 防幂等锁死 |
| `scripts/send_telegram.py` | TG 推送(所有模块共用);本地读 telegram配置.json,CI 读环境变量;--file 推任意文档 |
| `待审核/日期/` | 每日产物。**标准四件**:items.json / fetch_report.json / AI热点日报.md / 爆款选题榜.md |

注:`待审核/2026-07-10/` 里的 `AI串烧口播稿.md`、`今日精选6选题.md` 是当天会话的一次性产物,存档保留;此类产物今后不再放日报目录。

## 关键机制

- **抓取与整理分离**:Action 干确定性重活(云端 routine 沙箱无公网,只放行 github.com),routine 只读 items.json 做判断性整理——绝不让 Claude 读原始网页(烧额度)
- **幂等**:当日日报已存在→跳过;重跑需明确说"重跑"(--force)
- **失败安全**:单源失败不阻塞;>60% 源失败 exit 2,残缺数据存 items.failed.json,不写 items.json
- **选题榜**:红海惩罚保护差异化蓝海选题,不让全网大热点霸榜;每张选题卡带钩子标题/切入角度/适配写作skill/避坑

## 排障

- 某源 0 条:先看 fetch_report.json 是失败还是过滤后为空(微博热搜被社会新闻刷屏时 0 条正常)
- 源挂了:reference/sources.md 查坑;配置.json 可单独关掉该源
- 词表误放/漏放:改 配置.json 的 keywords_zh/keywords_en(英文是正则,短词带边界)
