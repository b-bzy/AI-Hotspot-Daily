## 关键机制

- **评审是诚实的**:score<8.5 满 3 轮就如实标注 met_threshold: false 交付当前最优,不硬凑分(07-12 首跑即 7.8 分诚实交付)
- 修订是**局部的**:按 Top 问题清单改,不重写全文,保结构保数据保论点
- 三个 skill 的规范文件是从 Jacob 的迁移包逐字提取的——**改进写作风格就改这些 reference 文件**,云端下次运行自动生效
- 早报未就绪:等 5 分钟重拉,最多 3 次,仍无则跳过当日(不编造)
